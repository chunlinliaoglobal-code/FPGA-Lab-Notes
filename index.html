💻 (1) Verilog 資料處理模型 (Data Flow Model)

在此部分，我們將使用 assign 關鍵字來描述電路邏輯。這是一種描述訊號流向的建模方式。

module your_design (
    input A, B, C,
    output Y
);
    // 使用 Data Flow Model (assign) 描述圖一電路
    assign Y = (A & B) | (~C); 
endmodule

📊 (2) University Program VWF 功能驗證

在燒錄到硬體之前，我們先透過波形模擬確認邏輯是否正確：

  • 建立一個新的 Vector Waveform File (.vwf)。
  • 匯入 Nodes 並設定輸入訊號的時序組合。
  • 執行 Functional Simulation 並檢查輸出結果是否與真值表相符。

💡 Pin Planner 操作步驟：
1. 點擊功能表 Assignments -> Pin Planner。
2. 在 Location 欄位手動輸入上述對應腳位。
3. 關閉 Pin Planner，並重新點擊 Start Compilation 進行完整編譯。

🚀 燒錄五大詳細步驟

Step 1. 全編譯 (Full Compilation)
設定完 Pin Planner 後，點擊工具列的「藍色實心箭頭」。這會將腳位資訊寫入編譯檔案。請確保左下角的進度條全部跑完且無 Error。
Step 2. 開啟 Programmer
點選功能表 Tools -> Programmer。確認右上方 Hardware Setup 顯示為 USB-Blaster [USB-0]。
Step 3. 偵測晶片 (Auto Detect)
點擊左側 Auto Detect，選擇 EP4CE22。你會在視窗中看到一塊晶片的圖示。
Step 4. 掛載與勾選檔案
  1. 選取畫面中的晶片圖示，點擊左邊 Change File。
  2. 進入 output_files 資料夾，選取編譯生成的 .sof 檔案。
  3. 極重要： 勾選中間表格的 Program/Configure 核取方塊。
Step 5. 執行下載 (Start)
點擊左側 Start。觀察上方進度條顯示 100% (Successful)。此時開發板上的 CONF_DONE 指示燈會亮起。
⚠️ 除錯提醒 (Troubleshooting)：
  • Hardware Setup 是空的：請檢查 USB 線是否插好，或重新安裝 USB-Blaster 驅動。
  • Start 按鈕是灰色的：通常是因為忘記勾選 Program/Configure 框。
  • 燈號反應不對：檢查 Pin Planner 是否漏按了 Enter 導致腳位沒存進去。

✅ 最終功能驗證

燒錄完成後，請直接操作開發板上的按鈕或指撥開關，觀察 LED 是否按照預期邏輯點亮。若符合真值表，即代表硬體驗證成功！
